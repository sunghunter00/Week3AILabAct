    Rule-based systems need override conditions because some rules are more important than others. In this activity, a teacher's pass should always be checked first because it automatically allows a student to use the fast lane. If Aliah checks the time and number of items before the teacher's pass, the program might reject someone who should actually be approved. This would give the wrong result and make the system unfair. Checking the override condition first makes sure the rules are followed correctly. It also makes the program easier to maintain if more rules are added later.
